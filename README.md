# THE K

리니지 클래식 전용 유틸리티 웹앱. React 19 + TypeScript(Strict) + Vite + Tailwind CSS 기반이며,
별도 백엔드 없이 브라우저 LocalStorage에 모든 데이터를 저장합니다.

이 저장소는 **완전히 독립된 프로젝트**입니다. 이전 버전이나 다른 파일에 의존하지 않으며,
이 ZIP 하나만 있으면 그대로 실행/빌드/배포할 수 있습니다.

**현재 버전: v1.0.0**

---

## 처음 실행하기

### 1) 요구 사항

- Node.js 18 이상 (권장: 20 이상)
- npm 9 이상

### 2) 설치 및 실행

```bash
npm install
npm run dev
```

터미널에 뜨는 주소(기본 `http://localhost:5173`)를 브라우저로 열면 바로 사용할 수 있습니다.
최초 실행 시 아이템 비교 DB에 활 5종(사이하의 활/화염의 활/장궁/크로스 보우/달의 장궁)이
기본으로 채워져 있습니다.

### 3) 프로덕션 빌드

```bash
npm run build
```

- `tsc -b`로 타입체크 후 `vite build`로 `dist/` 폴더에 프로덕션 번들을 생성합니다.
- 타입 오류가 있으면 빌드가 중단됩니다 (오류 0개가 정상입니다).
- 빌드 결과만 미리 보려면:
  ```bash
  npm run preview
  ```

### 4) 스크립트 전체 목록

| 명령 | 설명 |
|---|---|
| `npm run dev` | 개발 서버 실행 |
| `npm run build` | 타입체크 + 프로덕션 빌드 (`dist/` 생성) |
| `npm run preview` | 빌드 결과를 로컬에서 미리보기 |
| `npm run lint` | ESLint 검사 |

## GitHub Pages 배포 방법

1. 이 저장소를 GitHub에 그대로 업로드합니다 (이 ZIP을 풀어서 나온 내용 전체를 커밋).
2. `npm run build`로 `dist/` 폴더를 생성합니다.
3. `dist/` 폴더 내용을 `gh-pages` 브랜치에 올리거나, GitHub Actions로 자동 배포합니다.
   - 가장 간단한 방법: `npm i -g gh-pages` 후 `gh-pages -d dist`
   - 또는 저장소 Settings → Pages에서 GitHub Actions 배포를 사용합니다.

`vite.config.ts`에 상대경로(`base: './'`)가 이미 설정되어 있어서, 루트 사이트
(`사용자명.github.io`)든 프로젝트 사이트(`사용자명.github.io/저장소명/`)든 **별도 설정 없이**
그대로 배포하면 정상 동작합니다 (자산 경로가 깨지는 "흰 화면" 문제 없음).

## 데이터는 어디에 저장되나요?

모든 데이터는 **사용자 브라우저의 LocalStorage**(`theKData` 키)에만 저장됩니다. 서버로 전송되는
개인 데이터는 전혀 없습니다. 브라우저 저장소를 지우거나 다른 기기/브라우저로 넘어가면 데이터가
보이지 않으니, 아래 백업 기능을 꼭 활용하세요.

---

## 백업 / 복원 사용법

### 전체 백업 / 복원 (설정 화면)

**설정 → 데이터 관리**에서:
- **"전체 백업 (.thek)"** 버튼 → 모든 데이터(제작견적/경험치/24시간계산기/즐겨찾기/아이템/
  아데나매입/설정)를 파일 하나(`.thek`)로 내려받습니다.
- **"백업 파일 복원 (.thek)"** 버튼 → 그 파일을 다시 선택하면 전부 복원됩니다.
- 새 PC로 옮기거나, 브라우저를 바꾸거나, 만약을 대비해 주기적으로 백업해두는 걸 권장합니다.

### 기능별 내보내기 / 불러오기

경험치 / 아데나 매입 / 아이템 비교 / 제작 비교 견적 / 설정, 이 5개 화면 각각의 상단에
**📤 내보내기 / 📥 불러오기** 버튼이 있습니다.

- 그 화면의 데이터만 따로 `.thek` 파일로 저장하거나, 다른 길드원과 공유할 수 있습니다.
- 불러올 때 **다른 화면의 파일을 실수로 선택해도 안전하게 거부**됩니다 (예: "경험치" 화면에서
  "아데나 매입" 파일을 선택하면 "이 파일은 '아데나 매입' 데이터입니다"라고 알려주고 막습니다).
- 불러오기는 **그 화면의 데이터만 덮어쓰며, 다른 화면 데이터는 전혀 건드리지 않습니다.**

## OCR 사용법 (스크린샷 자동 인식)

경험치 기록, 아이템 등록 화면에서 게임 스크린샷을 붙여넣거나 업로드하면 숫자/텍스트를
자동으로 인식합니다.

1. 해당 화면에서 **"스크린샷으로 자동 채우기 (OCR)"** 버튼을 누릅니다.
2. 이미지를 업로드하거나 클립보드에서 붙여넣습니다 (Ctrl+V).
3. 처음 한 번은 숫자가 표시된 위치를 마우스로 드래그해서 지정해줍니다 (이후엔 자동 기억).
4. 인식된 값이 자동으로 입력창에 채워집니다 — **인식이 틀렸으면 직접 수정**하면 됩니다.
5. 경험치 화면에서는 레벨도 함께 인식을 시도합니다 (인식 실패 시 직접 입력).

인식률이 낮으면 스크린샷을 더 밝고 선명하게(확대) 찍어서 다시 시도해보세요.

## 아이템 비교 사용법

1. **검색 또는 새 아이템 등록**(OCR 또는 직접 입력)으로 아이템을 하나 선택/등록합니다.
2. 그 아이템과 **같은 무기 종류(계열)** 인 아이템 전체가 THE K DB에서 자동으로 검색되어
   비교표가 만들어집니다.
   - 예: "사이하의 활"을 선택하면 활 계열 전체(사이하의 활/화염의 활/장궁/크로스 보우/
     달의 장궁 등)가 자동으로 비교표에 채워집니다.
3. 비교표에는 **이미지 / 아이템명 / 공격력 / 원거리(또는 근거리) 명중 / 원거리(또는 근거리)
   대미지 / 특수옵션**만 표시됩니다.
4. 비교표는 JPG로 저장할 수 있고, 개별 아이템을 표에서 뺄 수도 있습니다.
5. THE K DB만 사용하며, 외부 사이트/크롤링은 전혀 쓰지 않습니다 — 검색 결과가 없으면
   OCR이나 직접 입력으로 새로 등록해서 DB를 채워나가면 됩니다.
6. 최초 설치 시 **활 5종이 기본 DB로 이미 들어있어**, 바로 비교 기능을 테스트해볼 수 있습니다.
   등록/수정/삭제는 자유롭게 가능합니다 (관리자 권한 구분 없이 누구나 DB를 관리합니다).

## 아데나 매입 사용법

1. **오늘 환율**과 **현재 수량(만)**을 입력하면 **"실시간 총 매입금액"**(환율 × 현재 수량)이
   입력 즉시 자동 계산됩니다.
2. **매입 등록**에서 아이디와 수량을 입력해 매입 기록을 하나씩 추가합니다. 비고(금액)는
   등록 시점의 환율 기준으로 자동 계산되어 저장됩니다.
3. **입금 여부 체크박스**는 "입금완료/미완료" 표시 전용입니다 — 어떤 계산에도 영향을
   주지 않습니다.
4. **"누적 매입 등록 금액"**은 지금까지 등록한 매입 기록들의 합계이며, "실시간 총
   매입금액"과는 다른 값입니다 (헷갈리지 않도록 카드 설명에 명시되어 있습니다).

## 경험치 사용법

1. **캐릭터 레벨**, **시작/종료 일시**(DateTime Picker, 기본값은 현재 시각), **시작/종료
   경험치(%)** 를 입력하고 저장합니다.
   - 레벨은 마지막으로 저장한 값을 자동으로 이어받습니다.
   - 시작 경험치는 직전 기록의 종료 경험치를 자동으로 이어받습니다 (연속 사냥 입력 편의).
2. 경험치 분석 화면에서 **레벨별 필요경험치 테이블 없이, 오직 %만으로** 다음을 확인합니다:
   - 현재 레벨/경험치, 남은 %
   - 최근 24시간 / 7일 평균 / 30일 평균 / 전체 평균 획득 속도
   - 각 구간 기준 100% 도달까지 남은 일수와 예상 완료 날짜
   - 목표일을 직접 입력하면 필요한 하루 평균 속도와 달성 가능 여부, 부족한 만큼의
     추가 필요 속도를 알려줍니다
   - 최고/최저 하루 기록, 최고 주간/월간 평균, 증가·감소 추세
   - 일별 획득량 그래프(막대) + 7일/30일 이동평균선 + 예상 도달 시점
3. 기록 목록에서 **날짜 범위 필터**, **검색**, **CSV 내보내기**를 사용할 수 있습니다.
4. **OCR로 인식한 경험치/레벨도 수동 입력과 완전히 동일하게** 모든 통계·그래프·목표
   계산에 포함됩니다.

React 19 · TypeScript(Strict, `noUncheckedIndexedAccess`) · Vite · Tailwind CSS ·

## 알려진 스코프 제한

- 드롭다운은 네이티브 `<select>`를 사용합니다 (검색 가능한 콤보박스는 미구현).
- 아이템 DB는 사용자가 직접 입력하거나 OCR로 등록한 데이터(+ 기본 제공 활 5종)만
  존재합니다. 외부 아이템 데이터베이스와는 연동하지 않으므로, 등록되지 않은 아이템은
  검색되지 않습니다.
- 아데나 매입의 "오늘 환율"은 사용자가 직접 입력하는 값이며, 실시간 시세 자동 연동은 없습니다.
